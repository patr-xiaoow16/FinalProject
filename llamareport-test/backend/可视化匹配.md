# 年报文字内容 × 可视化视图推荐规则文档

---

## 一、文档目的

本文件用于指导系统（LLM、规则引擎、前端可视化模块）在解析**年报中的文字内容**时，判断：

1. 该段文字在年报阅读流程中承担的角色  
2. 年报读者在该位置最可能提出的问题  
3. 是否值得生成可视化视图  
4. 若生成，应选择哪一类视图  

---

## 二、核心原则

> **是否可视化 ≠ 是否有数字**  
> **是否可视化 = 是否能帮助年报读者形成判断**

只要一段年报文字能够：
- 帮助判断好坏
- 解释变化原因
- 描述结构关系
- 揭示风险或不确定性

即使**完全没有财务数据**，也**允许并且应该**进行可视化。

---

## 三、年报文字问题类型定义

系统需先将年报文字归类为以下类型之一：

- 数据类  
- 核心结论类  
- 结构描述类  
- 过程与变化类  
- 风险与不确定性类  
- 态度与语气类  
- 跨年度对比类  
- 纯合规 / 事实类（不生成视图）

---

## 四、视图推荐规则表（核心表）

| 问题类型 | 年报读者最可能问的问题 | 推荐视图 |
|---|---|---|
| 数据类 | 指标现在处于什么水平？<br>趋势是上升还是下降？<br>是否存在异常或拐点？<br>数据是否支持文字表述？ | 折线图<br>柱状图<br>KPI 指标卡<br>异常点标注 |
| 核心结论类 | 这一年整体经营是好是坏？<br>管理层最重要的判断是什么？<br>与去年相比是否变化？<br>语气是乐观还是谨慎？ | Insight Card<br>状态标签 |
| 结构描述类 | 公司靠什么赚钱？<br>哪块业务最核心？<br>是否存在结构性风险？ | 桑基图<br>业务结构图<br>价值链示意图 |
| 过程与变化类 | 这一年发生了哪些关键事件？<br>事情发生在什么时候？<br>推进是否连续？ | 时间轴（Timeline）<br>阶段流程图 |
| 风险与不确定性类 | 哪些风险是新增的？<br>哪些被反复强调？<br>风险若发生先影响哪里？ | 风险矩阵（定性）<br>风险趋势标签 |
| 态度与语气类 | 管理层对未来偏乐观还是谨慎？<br>是否出现模糊或回避？ | 词云图 |
| 跨年度对比类 | 今年新增强调了什么？<br>去年内容是否被弱化？ | 文本 Diff 视图<br>关键词变化热度 |
| 纯合规 / 事实类 | （通常不会引发思考） | 不生成视图 |

---

## 五、视图生成决策规则（If–Then）

### 规则 1：是否生成视图
IF 问题类型 == 纯合规 / 事实类
THEN 不生成任何视图
ELSE 进入视图推荐流程


### 规则 2：非数据文字也允许生成视图（关键规则）
IF 问题类型 IN [
  核心结论类,
  结构描述类,
  过程与变化类,
  风险与不确定性类,
  态度与语气类,
  跨年度对比类
]
THEN 允许生成非数值型可视化视图
